Historias Fase 1 - Registro de usuarios


Las historias fase 1 deben reflejar los requerimientos más generales, esos objetivos principales por los que se va a crear el App.


Historia de usuario #1
Identificador (ID) de la Historia:
00-0000-0001
Rol:
Como un Analista de Negocio
Característica / Funcionalidad:
Necesito un modulo para el registro de usuarios en el app
Razón / Resultado:
Con la finalidad de tener un sistema seguro donde solo van a poder acceder usuarios registrados en el sistema
Número (#) de Escenario:
1
Criterio de Aceptación (Título):
Registro de Usuario
Contexto:
En caso que el usuario se registre en el sistema debe llegarle una notificacion al correo electrónico con la confirmación de su registro
Evento:
Cuando el usuario se registre en el sistema
Resultado / Comportamiento esperado:
El sistema debe enviar notificacion al correo del usuario de la confirmación de su registro y este usuario debe esperar otra notificación confirmando que la validación de datos es exitosa o errónea

Identificador (ID) de la Historia:

Rol:
Como un Analista de Negocio
Característica / Funcionalidad:
Necesito la notificacion al correo electrónico del resultado de validación de datos
Razón / Resultado:
Con la finalidad de continuar/finalizar con el proceso de inscripción en el app
Número (#) de Escenario:
1
Criterio de Aceptación (Título):
Confirmación exitosa en la validación de datos
Contexto:
En caso que la confirmación de datos sea exitosa el usuario ya puede ingresar al sistema
Evento:
Cuando la confirmacion de datos sea exitosa
Resultado / Comportamiento esperado:
El sistema debe dejar al usuario dar la opción de iniciar sesión
Número (#) de Escenario:
2
Criterio de Aceptación (Título):
Confirmación errónea en la validación de datos
Contexto:
En caso que la confirmación de datos sea errónea el usuario no podra ingresar al sistema
Evento:
Cuando la confirmacion de datos sea errónea
Resultado / Comportamiento esperado:
El sistema no debe dejar al usuario dar la opción de iniciar sesión

2

Evento:
Cuando el usuario ingrese credenciales de manera incorrecta
Resultado / Comportamiento esperado:
El sistema no debe dejar ingresar al usuario y debe retornar un error estandar para que el usuario pueda volver a digitar sus credenciales
